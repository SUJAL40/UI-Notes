									CSS Z-Index
									
- It controls the placement of overlapping contents in page.
- You can send backward or back.(backword one step back, back means behind all)
- You can bring forward or front.(forward means one step forward , front on top of all)

Syntax:

							CSS Background Properties 

- Background properties are used to control the background of an element.

	background-color → sets color						: color
	background-image → sets image						: url()
	background-size → controls image size				: contain, cover, [width, height]
	background-position → controls image position		: top, center, bottom, left, right [x, y]
	background-repeat → controls repetition				: no-repeat, repeat, repeat-x, repeat-y
	background-attachment → controls scrolling behavior : fixed, scroll
	background-clip → controls background area			: content-box, padding-box

- background-clip
	| Value         | Meaning                                   |
	| ------------- | ----------------------------------------- |
	| `border-box`  | Background extends under border (default) |
	| `padding-box` | Background stops at padding               |
	| `content-box` | Background only inside content area       |

								CSS Display

- Display can control various display styles for element in page.

	display:none	: 	It hides the element in page and removes the allocated
						space on page.

FAQ: What is difference between display:none & visibitily:hidden ?
Ans: display:none removes the allocated space after hiding.
	 visibility:hidden keeps the allocated space.

	display:block   :	It will not allow to wrap any content beside.
	display:inline	:	It allows to wrap the content beside.


Q) When should we use inline and when should we use flex?

A) inline is used when an element should take only the required width and stay in the same line.
   flex is used on a parent container to align and arrange its child elements horizontally or vertically.
	
Note: Block allows to modify the size and position of element. It will not allow content beside.

	display:inline-block	: It allows to modify size and keep inline.
	
display: inline-block in CSS
inline-block combines the features of inline and block elements.
Behavior
- Appears on the same line (like inline)
- Allows width, height, margin, and padding (like block)

								Display Flex

- It is used for container that comprises of elements.
- Elements in container can be displayed in 2 orientations
	a) Row
	b) Column
- Flexible row contains elements with alignment that is responsive with regard to screen
and parent.

Syntax:
	{
		display:flex;
		flex-direction : row | column;
	}

- Flex wrap will control the wrapping of elements.

Syntax:
	{
		flex-wrap: warp | no-wrap;
	}

Q) flex-wrap: wrap; in CSS
A. flex-wrap is used with Flexbox. It decides whether flex items should stay in one line 
   or move to the next line when there isn't enough space.
  
Q: What is flex-wrap: wrap?
A: flex-wrap: wrap allows flex items to move onto multiple lines when there isn't enough space
   in the flex container. Without it, all flex items try to stay on a single line
  (nowrap is the default behavior).
  
- Alignment can be defined using

	a) justify-content
	b) align-items

Syntax:
	{
		justify-content: space-between | space-around | space-evenly | center;
		align-items: start | end | center | baseline
	}

- align-items row-wise orientation keeps content top, center or bottom.
- align-items column orientation keeps left, center, right.

					top, left		: baseline
					center			: center
					right, bottom	: end

						Display Grid

- It is used to create responsive layout.
- You can change the placement of any content using grid-row or grid-column.
- Grid allows dynamic placements using media queries.
- You can change the complete page layout according to the device.

Attribute

| Property                | Purpose                         |
| ----------------------- | ------------------------------- |
| `display: grid`         | Creates a grid container        |
| `grid-template-columns` | Defines column sizes            |
| `grid-row`              | Controls row placement/span     |
| `grid-column`           | Controls column placement/span  |
| `gap`                   | Adds spacing between grid items |

- Media Queries are required to change the layout dynamically.

	@media type and (attributes)

type		=> screen, print, speech
attributes	=> orientation, width, min-width, max-width

Syntax:
	@media screen and (orientation:landscape)

						CSS Transforms
						
- Transform allows to change the position, size and orientation of element.
- Without disturbing existing content of page if we control element then use transform.
- Transform provides
	a) 2D [2 Dimensional] methods
	b) 3D [3 Dimensional] methods

Transform 2D:
- It controls element along X & Y axis.

	a) translate()
	b) scale()
	c) rotate()
	d) skew()
	e) matrix()

| Function      | Purpose                | Example                 | Easy Memory          |
| ------------- | ---------------------- | ----------------------- | -------------------- |
| `translate()` | Move element           | `translate(50px, 20px)` | **Transport = Move** |
| `scale()`     | Increase/Decrease size | `scale(2)`              | **Scale = Zoom**     |
| `rotate()`    | Rotate element         | `rotate(45deg)`         | **Rotate = Turn**    |
| `skew()`      | Tilt element           | `skew(20deg)`           | **Skew = Slant**     |
| `matrix()`    | Combine all transforms | `matrix(1,0,0,1,50,20)` | **Master Transform** |


							Translate:
							
- It moves the element along X and Y axis.

	a) translate()
	b) translateX()
	c) translateY()

Syntax:
	{
		transform: translate(200px, 100px);
	}

								Keyframes & Animations


Keyframes & Animations
- Keyframe is a container with element that changes according to state and situation.
- Animation is created with a set of keyframes.
- Keyframe animation comprises of
	a) Initial State
	b) Final State
	c) Steps

Syntax:

	@keyframes	TransitionName
	{	
		from { }
		50% { }
		to { }
	}

- Keyframes are assigned to any element by using animation properties

animation-name
animation-duration
animation-property
animation-delay
animation-direction
animation-iteration-count
animation-timing-function

| Property                      | Purpose                                        | Interview Answer                                                    | Real Developer Usage                              | Example                                   |
| ----------------------------- | ---------------------------------------------- | ------------------------------------------------------------------- | ------------------------------------------------- | ----------------------------------------- |
| **animation-name**            | Defines which `@keyframes` animation will run. | Connects an element with a keyframe animation.                      | Used when multiple animations exist in a project. | `animation-name: slide;`                  |
| **animation-duration**        | Sets animation running time.                   | Specifies how long an animation takes to complete one cycle.        | Common values: `0.3s`, `0.5s`, `1s`, `2s`.        | `animation-duration: 1s;`                 |
| **animation-delay**           | Waits before animation starts.                 | Delays the start of an animation.                                   | Useful for staggered effects and loaders.         | `animation-delay: 2s;`                    |
| **animation-direction**       | Controls animation movement direction.         | Determines whether animation runs forward, backward, or alternates. | Creates smooth back-and-forth effects.            | `animation-direction: alternate;`         |
| **animation-iteration-count** | Number of times animation repeats.             | Specifies how many times animation runs.                            | Loaders often use `infinite`.                     | `animation-iteration-count: infinite;`    |
| **animation-timing-function** | Controls animation speed curve.                | Defines speed variation during animation.                           | Makes animation feel natural.                     | `animation-timing-function: ease-in-out;` |
| **animation-fill-mode**       | Controls styles before/after animation.        | Decides whether keyframe styles persist.                            | Useful for keeping final state.                   | `animation-fill-mode: forwards;`          |
| **animation-play-state**      | Starts or pauses animation.                    | Controls running or paused state.                                   | Often used on hover.                              | `animation-play-state: paused;`           |

Syntax:
	element {
		animation-name: KeyFrameName;
		animation-duration: 2s;
		animation-iteration-count: 2 | infinite;
		animation-direction: reverse | alternate;
		animation-timing-function: ease-in;
	}

							JavaScript

- JavaScript is light weight interpreted and JIT compiled programming language.
- Light weight allows to run on device or in environment with less memory.
- Interpreted allows line by line translation.
- Compiler allows translation of all lines simultaneously at the same time.
- There are 2 types of compiling techniques
	a) JIT
	b) AOT
- JIT is "Just-in-Time", which compiles the code only when it is requested.
- AOT is "Ahead-of-Time", which compiles the code at application level before it is
requested.
- JavaScript is a language that supports various programming techniques and
approaches like
	a) Structural Programming
	b) Imperative Programming
	c) Functional Programming
	d) Object Oriented Programming etc ..

Note: JavaScript is not an OOP language. It supports only few features of OOP.

- JavaScript have various application areas, it is used

	a) Client Side		: HTML
	b) Server Side		: Node JS
	c) Database			: MongoDB
	d) Animation Tools	: CAD, Flash 2D, 3D etc ..

FAQ: What is the purpose of JavaScript client side?
Ans : A client side script is used to reduce burden on server.
It saves round trips.

FAQ: What are the activities managed by JavaScript client side?
Ans:
	1. Browser Interactions [BQM]
		- window
		- location
		- navigator
		- history
		- document
	2. DOM Manipulations
		- Adding elements
		- Removing elements
		- Databinding
		- Style Binding
		- Class Binding
		- Event Binding
	3. Client side Validations
		- Verifying user input
		- Restricting contradictory values
		- Authorizing values etc ..

Evolution of JavaScript:

	- CERN labs introduced ECMA Script, which is known as the base for all scripts in the
	  world.
	- Internet started with a browser called Mosaic, which uses ECMA script & HTML.

	- Netscape appointed "Brendan Eich" to develop a script for their browser.
	- He designed a script by name "Moca", later renamed as "Live Script".
	- Netscape given the responsibility of Live Script to Sun Micro Systems. [Java]
	- Sun Micro Systems and Netscape together name the script as "JavaScript".
	- In 2000 Netscape stopped its services and given the responsibility of JavaScript to

	- Latest version of JavaScript is ESNext [Alpha], the stable and standard versions used
	  in companies
	  
		ECMA 2015	ES5

Q) Difference between strictly typed and strongly typed ?
A. Strict typing means a variable must follow its declared data type.
   Strong typing means different data types cannot be mixed automatically.
   For example, in TypeScript a number variable cannot store a string, 
   and in Python you cannot add a string and a number without conversion.

- Javascript standard are maintained by tool ESLint.(It verify language standards)
- Bable is a js compiler

Issues with JavaScript:

- JavaScript is not strongly typed.
- No restriction for type of data to store in a reference.
	x = 10;		x is number
	X = "A";	x changes to string
	x = true;	x changes to Boolean
- You need lot of validations to restrict a value type.
- JavaScript is not implicitly strictly typed.
- It allows to write without following the rules of programming.
- It leads to lot of code inconsistency.
- It is not an OOP language.
- It is not easy to extend.
- It is not good in code level security.

Solution:
- TypeScript
- It is just an alternative for JavaScript and not replacement.
- Browser can understand only JavaScript and not TypeScript.
