	🌐 Web Concepts – Simple Notes

	🔹 1. What is a Website?

	A website is a collection of web pages stored on a server
	It acts like a virtual folder (directory) on a web server
	It mainly provides information (no interaction)

	👉 Example:
	Company website showing details
	Portfolio website

	👉 Key Point:
	Website = Static (just view, no input)

	🔹 2. What is a Web Application?

	A web application is an advanced website
	It allows user interaction (input + output)
	Users can perform actions like:
	Login
	Fill forms
	Upload data

	👉 Example:
	YouTube (upload, comment, like)
	Spotify (search, play, create playlist)

	👉 Key Point:
	Web Application = Dynamic (interaction allowed)

	🔹 3. Website vs Web Application (Quick Trick)

	FeatureWebsiteWeb ApplicationTypeStaticDynamicInteraction❌ No✅ YesExampleInfo pagesLogin systems
	👉 Exam Trick:
	Website → Read only
	Web App → Read + Write (Interact)

	🔹 4. Web Content Types
	The Web includes many types of content:
	Website
	Web Application
	Blog (articles/posts)
	Microblog (short posts like tweets)
	Wiki (editable content)
	Podcasting (audio/video content)

	🔹 5. Website Paths
	A website has 2 types of paths:
	a) Physical Path
	Actual location on server (folder/files)
	b) Virtual Path
	URL seen by user (friendly path)

	👉 Example:
	Physical: C:/website/index.html
	Virtual: www.site.com/index

	🔹 6. Static vs Dynamic Resources
	Static: Only view (HTML, images)
	Dynamic: Changes based on user input

	👉 Important:
	Website → Only static
	Web App → Static + Dynamic

	🔹 7. Podcasting

	Sharing audio/video episodes online
	Can listen anytime
	👉 Example:
	Spotify
	YouTube
	👉 Simple Meaning:
	Podcast = Radio on internet (anytime play)

	🔹 8. Wiki

	A wiki is a website where users can edit content
	👉 Example:
	Wikipedia
	Google Maps (user edits places)
	👉 Key Point:
	Wiki = User can edit content

	🔹 9. Logs (Important)
	Logs = record of activities
	Stores:
	User actions
	Errors
	Login info

	👉 Simple:
	Logs = History of what happened on system

	🔥 Final Revision Shortcut

	Website → Static (only view)
	Web App → Dynamic (interaction)
	Podcast → Audio/video episodes
	Wiki → Editable content
	Logs → Activity records


Creating a New Project Folder and Opening It in Visual Studio Code

	Step 1: Create a New Folder for Your Project
	Navigate to any location on your computer where you want to store your project.
	Create a new folder and give it a name, for example: travel-app.
	On Windows, you can do this by right-clicking in the desired location, selecting New > Folder, and typing the folder name.
	
	Step 2: Open Visual Studio Code
	Launch Visual Studio Code from your Start Menu, desktop shortcut, or application list.
	
	Step 3: Open the Folder in Visual Studio Code
	In Visual Studio Code, go to the File menu.
	Select Open Folder.
	Browse to the folder you created (e.g., D: travel-app) and click Select Folder.
	
	Step 4: Open a New Terminal in Visual Studio Code
	In Visual Studio Code, go to the Terminal menu.
	Select New Terminal.
	Alternatively, you can use the keyboard shortcut Ctrl + ` (backtick key, usually located under the Esc key).
	You now have your project folder open in VS Code and a terminal ready for running commands.

	🔹 Terminal / PowerShell

	A terminal is a tool where you type commands to control your computer (OS).
	In VS Code, you can switch between PowerShell and Command Prompt (CMD).

	🔹 npm init -y

	The command npm init -y creates a new project setup automatically.
	It generates a file called package.json.
	The -y means “yes to all questions” (auto setup).

	🔹 package.json

	package.json is a file that stores project information (metadata).
	It contains:
	Project name
	Version
	Description
	Every Node.js project should have this file.

	🔹 README.md
	
	README.md is a help/document file for developers.
	It explains how the project works and how to use it.

	🔹 Images for Development

	Do not take images randomly from Google (copyright issue).
	Use free image websites like:
	Pixabay
	Freepik

	🔹 index.html

	A website usually starts with index.html.
	It is the main page (homepage) of the website.

	🔹 Final Important Point

	Always check all data and files before starting or running your project.

	README → Guide
