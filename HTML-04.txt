Display Classes :

| Class                      | Notes                                  |
| -------------------------- | -------------------------------------- |
| `.d-none`                  | Hides the element (`display:none`)     |
| `.d-inline`                | Displays element inline                |
| `.d-block`                 | Displays element as block              |
| `.d-flex`                  | Enables flex container                 |
| `.d-inline-block`          | Inline element with block properties   |
| `.d-inline-flex`           | Inline flex container                  |
| `.flex-wrap`               | Allows flex items to move to next line |
| `.flex-row`                | Arranges flex items horizontally       |
| `.flex-column`             | Arranges flex items vertically         |
| `.justify-content-between` | Adds space between flex items          |
| `.justify-content-center`  | Centers items horizontally             |
| `.justify-content-end`     | Moves items to end horizontally        |
| `.justify-content-start`   | Moves items to start horizontally      |
| `.align-items-center`      | Centers items vertically               |
| `.align-items-start`       | Aligns items at top/start              |
| `.align-items-end`         | Aligns items at bottom/end             |

- Flexbox is used to align and distribute elements efficiently inside a container.

| Class             | Simple Meaning                                                                       |
| ----------------- | ------------------------------------------------------------------------------------ |
| `.d-inline-block` | Element stays in same line like inline, but you can give width and height like block |
| `.d-inline-flex`  | Element stays in same line and also becomes a flex container                         |
| `.flex-wrap`      | If flex items do not fit in one line, they move to next line                         |



Ex.

	<div class="m-4 p-4 d-flex flex-row w-50 justify-content-between">
		<button>A</button>
		<button>B</button>
		<button class="d-inline">C</button>
		<button>D</button>
		<button>E</button>
	</div>

2. Modal
	- It is dialog that pops up.
	- Content is in the backdrop and dialog is over the content.

Classes:

| Class                      | Detailed Explanation                                               |
| -------------------------- | ------------------------------------------------------------------ |
| `.modal`                   | Main popup container. Creates backdrop and hidden popup structure. |
| `.modal-dialog`            | Controls popup size and position.                                  |
| `.modal-content`           | Wrapper containing header, body, footer.                           |
| `.modal-header`            | Top area for title and close button.                               |
| `.modal-body`              | Main content area of modal.                                        |
| `.modal-dialog-centered`   | Vertically centers modal on screen.                                |
| `.modal-dialog-scrollable` | Adds scrollbar inside modal body if content is large.              |
| `.modal-fullscreen`        | Opens modal in full screen.                                        |
| `.modal-footer`            | Bottom section for buttons/actions.                                |


Syntax:

<div class="modal">
	<div class="modal-dialog">
		<div class="modal-content">
			..header, body ,footer
		</div>
	</div>
</div>

- Every modal must have an "ID" reference.
- Every modal is hidden in page.
- You can open specific model with button or link click.
- It requires attributes.

Dynamic Attributes

	data-bs-toggle	: which component to open, it refers to component name
	data-bs-target  : It refers to component "ID"
	data-bs-dismiss	: which component to close

Ex.

<!-- Modal container -->
<!-- class="modal" = creates Bootstrap modal popup -->
<!-- id="login" = unique modal identifier -->
<div class="modal" id="login">

</div>


<!-- Button to open modal -->
<!-- data-bs-toggle="modal" = tells Bootstrap to open modal -->
<!-- data-bs-target="#login" = opens modal whose id is login -->
<button data-bs-toggle="modal"
        data-bs-target="#login">

    Open Modal

</button>


<!-- Anchor tag to open modal -->
<!-- href="#login" = points to modal id -->
<!-- data-bs-toggle="modal" = opens modal popup -->
<a href="#login"
   data-bs-toggle="modal">

    Open Login Modal

</a>


- #login means:	id="login"	# is used to target element id.

Syntax:
	<div class="modal" id="login"> </div>

	<button data-bs-toggle="modal" data-bs-target="#login">
	<a href="#login" data-bs-toggle="modal">
	

- To open model with transition use ".fade" class.

	<div class="modal fade"> </div>
	
- data-bs-toggle="modal" : Defines modal action on click.
- data-bs-target="#login" : Targets and opens modal having id="login".

4. Accordion 

- It is a combination of multiple items using "Mutex" mechanism. 
- You can select one item to open and show, where all other items will close.
- Accordion is a group of collapsible items.
- Only one item stays open at a time.
- When one item opens, other items close automatically.
- This mechanism is called : Mutex (Mutual Exclusion)

| Class                 | Notes                               |
| --------------------- | ----------------------------------- |
| `.accordion`          | Main accordion container            |
| `.accordion-item`     | Single accordion item               |
| `.accordion-header`   | Header/title section                |
| `.accordion-body`     | Main content area                   |
| `.accordion-collapse` | Expand/collapse container           |
| `.accordion-button`   | Clickable button to open/close item |


Syntax:
	<div class="accordion" id="">
		<div class="accordion-item">
		... header, body with collapse ...
		</div>
		<div class="accordion-item">
		
		</div>
	</div>

Attribute:
	data-bs-target
	data-bs-toggle
	data-bs-parent	: It is defined for all accordion-collapse items.
					  It refers to ID of accordion.

	<div class="accordion-collapse collapse" id="q1" data-bs-parent="#faqs">

- It configures sliding and fading content in page.

| Class             | Notes                             |

	<div class="carousel" data-bs-ride="carousel">
	
- Carousel can have 2 types of animation styles

		.carousel-fade
		.slide

	<div class="carousel slide">
	<div class="carousel carousel-fade">

								Styles in CSS
								
- Styles provide a set of attributes to make HTML more interactive and responsive.
- Style uses an object to refactor the code.
- Object contains relative data and logic.
- Style objects are used to make HTML interactive.
- (attribute value are defined with colon " : " eg color : red )
- Styles can defined for HTML elements in 3 ways

	a) Inline Styles
	b) Embedded Styles
	c) External Style Sheet

Inline Styles:
- The style object is defined for every element individually using "style" attribute.
- It is faster in rendering.
- It is used when we give unique style for any one element
- It is not good for reusability.

Syntax:
	<h2 style="attribute:value; attribute:value"> </h2>

Eg.
	<h2 style="color:red; font-family:Arial"> HTML </h2>


Embedded Styles:

- In embedded technique the style object is defined in a <style> container.
- So that it is good for reusability and separation.
- Embedded is good for extensibility.
- "Good for extensibility" means:
- You can easily add more CSS rules and styles for many elements within the same page.

Syntax:

<style>
	selector
	{
		attribute:value;
		attribute:value;
	}
</style>

- It is slower than inline.
- You can't access embedded styles from another page.
- Embedded is only for the context of current page.

FAQ's:
1. Where to embed the <style> container, in <head> or in <body> ?
A. You can embed in both sections.

2. What is difference between styles in head and styles in body?
A. Styles in head are used to save round-trip.
   Styles in body are for one time rendering.
   
   Meaning of "save round-trip":
   CSS is loaded along with the HTML page itself.
   No separate CSS file request is needed.
   So the browser doesn't need an extra trip (request) to fetch a CSS file.

3. If styles are defined for element in both head and body then which set of attribute will
   apply?
A. If different attributes are used then all will apply.
   If same attribute is used then the latest will apply. [LIFO]
   means if you write style for element in head and body then latest is body then 
   body section style will apply

- this is not official 
- if you write style tag outside the html section then it work as it is but it is not recommended

4. What is the media type for embedded styles?
A. Media type is used to define style for various media devices, which include
	a) screen
	b) print
	c) speech

Syntax:
	<style media="all | screen | print | speech">

	</style>

							Selectors in Style Object
							
- Object is a key and value collection.
- It keeps all related data and logic together.
- Style object is applied to HTML element using selectors.

Syntax:

	selector
		{
			color:red;
			text-align:center;
		}

- CSS provides various types of selectors to select HTML elements.
- We can categorize them into following groups

	1. Primary Selectors
	2. Rational Selectors
	3. Attribute Selectors
	4. Dynamic Pseudo classes
	5. Structural Pseudo classes
	6. Element state pseudo classes
	7. Validation state pseudo classes
	8. Universal selector
	9. Language selector
	10. Root selector etc ..

Primary Selectors
	1. Type Selector
	2. Id Selector
	3. Class Selector

# Type Selector:

- It refers to element token name.
	h1, p, div, span, table, td, dt, etc ..
- Type selector are good to apply at every occurrence of element in page.
- You can't skip at any specific occurrence.
	
 Syntax:
 
  h1{
	color : red;
  }

# ID Selector:

- Every element can have a reference "Id" [Identification]
- CSS can access the ID using "#" reference and can define effects.
- You can apply style to specific element.
- HTML element can't have multiple "Id" references.
- Hence you can't apply multiple categories of styles to one element using ID selectors.

Syntax:
	#text-style
	{
		color:red;
	}

	<div id="text-style">
	<h1 id="text-style">

# Class Selector:

- It is defined by using "." and reference name.
- It is accessed and applied to element using "class" attribute.
- You can apply multiple classes to one element.

Syntax:
	.text-style { }
	.bg-style {}

	<div class="text-style bg-style">

FAQ: If styles are applied to element using id, type & class then which set will apply?
Ans: According to priority if same attributes are used.
	1st	: Id
	2nd	: Class
	3rd	: Type
	It can apply all if different attributes are used.

Ex.

<style>
	h1 {
		color: red;
		border:1px solid red;
	}
	
	#text-style {
		color: green
	}

	.effect {
		color: blue
		padding: 20px;
	}
</style>

<body>
	<h1 class="effect" id="text-style">Web Technologies</h1>	
</body>

									CSS Inheritance
									
- Inheritance is a mechanism where the parent styles are auto implemented for child
  elements. However all parent features are not inherited.
- Developer can control the inheritance of any style by using following values

	a) inherit	: It inherit the parent style
	b) initial	: It remove inherited style

| ---- | ------------------ | ---------------------------------------- | --------------------------- |

| `q`  | Quarter-millimeter | 1/4 of a millimeter                      | `1q = 0.25mm ≈ 0.945px`     |




- Relative units are not defined with a fixed value it changes according to the parent.

| Position   | Easy Meaning                     | Memory Trick           |
| ---------- | -------------------------------- | ---------------------- |
| `static`   | Normal position (default)        | **Stay where you are** |
| `relative` | Move from your original position | **Move yourself**      |
| `absolute` | Move relative to parent          | **Follow parent**      |
| `fixed`    | Stay fixed on screen             | **Stick to screen**    |
| `sticky`   | Scroll normally, then stick      | **Scroll → Stick**     |

- 10-Second Interview Answer

CSS provides five position values:

	Static → Default position.
	Relative → Moves from its original position.
	Absolute → Positioned relative to its parent.
	Fixed → Fixed to the browser screen.
	Sticky → Scrolls normally and then sticks during scrolling.
	
Super Short Memory Formula

	Static   = Stay
	Relative = Move Yourself
	Absolute = Follow Parent
	Fixed    = Stick to Screen
	Sticky   = Scroll Then Stick

This formula is easy to remember and works well in interviews.


Here are some common **CSS Position Interview Questions** with short answers:

---

### 1. What is the default position of an HTML element?

✅ **Answer:**

> The default position is `static`.

---

### 2. How many position values are available in CSS?

✅ **Answer:**

> Five: `static`, `relative`, `absolute`, `fixed`, and `sticky`.

---

### 3. What is `position: relative`?

✅ **Answer:**

> It positions an element relative to its original position while keeping its original space reserved.

---

### 4. What is `position: absolute`?

✅ **Answer:**

> It positions an element relative to its nearest positioned parent and removes it from the normal document flow.

---

### 5. What is `position: fixed`?

✅ **Answer:**

> It positions an element relative to the browser viewport and keeps it visible during scrolling.

---

### 6. What is `position: sticky`?

✅ **Answer:**

> It behaves like `relative` initially and becomes fixed when a specified scroll position is reached.

---

### 7. What is the difference between `relative` and `absolute`?

✅ **Answer:**

> `Relative` keeps its original space and moves from its original position.
> `Absolute` removes the element from the normal flow and positions it relative to its parent.

---

### 8. What is the difference between `absolute` and `fixed`?

✅ **Answer:**

> `Absolute` is positioned relative to its parent.
> `Fixed` is positioned relative to the browser window and does not move on scroll.

---

### 9. What is the difference between `sticky` and `fixed`?

✅ **Answer:**

> `Fixed` is always fixed on the screen.
> `Sticky` becomes fixed only after reaching a specified scroll position.

---

### 10. Which position value is used for a WhatsApp chat button?

✅ **Answer:**

> `position: fixed`.

---

### 11. Which position value is used for a notification badge on an icon?

✅ **Answer:**

> `position: absolute`.

---

### 12. Why do we use `top`, `left`, `right`, and `bottom`?

✅ **Answer:**

> They are used to move positioned elements (`relative`, `absolute`, `fixed`, `sticky`).

---

### 13. Do `top`, `left`, `right`, and `bottom` work with `position: static`?

✅ **Answer:**

> No, they do not work with `static`.

---

### 14. What happens if an absolute element has no positioned parent?

✅ **Answer:**

> It is positioned relative to the page (the initial containing block).

---

### 15. Give one real-world example of each position.

| Position | Example                 |
| -------- | ----------------------- |
| Static   | Normal paragraph        |
| Relative | Slightly shifted button |
| Absolute | Notification badge      |
| Fixed    | WhatsApp/chat button    |
| Sticky   | Sticky navbar/header    |

---

### Interview Rapid-Fire

**Q:** Default position?
**A:** Static

**Q:** Which position follows the parent?
**A:** Absolute

**Q:** Which position sticks to the screen?
**A:** Fixed

**Q:** Which position scrolls first and then sticks?
**A:** Sticky

**Q:** Which position keeps its original space?
**A:** Relative

These are exactly the kind of questions frequently asked in HTML/CSS fresher interviews.
