FAQ's:

5. How to define one set of statements for multiple cases?
A. By defining the cases line-by-line

	case 1:
	case 2:
	case 3:
		statements;
		break;

eg.

<script>
    function f1() {
        var product = prompt("Enter Product Name");
        switch (product) {
            case "TV":
            case "Mobile":
            case "Watch":
                document.write(`${product} belongs to Electronics category`);
                break;
            case "Casuals":
            case "Boots":
                document.write(`${product} belongs to Footwear category`);
                break;
            default:
                document.write(`Avail Products : TV, Mobile, Watch, Casuals, Boots only..`);
                break;
        }
    }
    f1()
</script>


- If you case is refering to one character then you can write multiple cases like for Y and N : Y,y,n,N etc
- but for string we use to lowercase()

eg.

<script>
    function f1() {
        var choice = prompt("Enter Choice Yes or No");
        switch (choice.toLowerCase()) {
            case "yes":
                document.write("You selected yes to continue");
                break;
            case "no":
                document.write("You selected No to stop");
                break;
            default:
                document.write("Please enter your choice Y or N");
                break;
        }
    }
    f1();
</script>

6. How you write case for range of values?
A. By using Boolean expression that verifies the range of values.
   IF case is using a Boolean expression then the switch must always use true as value.

Syntax:

	switch(true)
	{
		case booleanExpression:
			statements;
			break;
	}
	
- When we are writing cases with boolean value or expression , switch value must be true eg.switch(true)

eg.

<script>
    function f1() {
        var n = parseInt(prompt("Enter Number"));
        switch (true) {
            case (n >= 1 && n <= 10):
                document.write(`Your number ${n} is between 1 to 10`);
                break;
            case (n >= 11 && n <= 20):
                document.write(` Your number ${n} is between 11 to 20 `);
                break;
            default:
                document.write("Pleae enter a number between 1 to 20 only");
                break;
        }
    }
    f1();
</script>

7. Can we define case without jump statement?
A. Yes
	Yes. Switch will continue to execute next case until break occurs.
	If there is no break in next level cases then it executes up to end, which include the
	default.

- We can use this in offer prices like buy 1 get 1 free , buy 2 get 1 free

eg.

                document.write("Collect your three<br>");
                break;

                document.write("Please enter a number between 1 to 3 only");
                break;

								Array Manipulations

1. Reading Array Elements

	toString()		: It returns all array elements with "," delimiter.
	join()			: It returns all array elements with custom delimiter.
	map()			: It is an iterator that reads elements in sequential order.
	forEach()		: It is an iterator that reads elements and index in sequential order

Ex:
	<script>
		var categories = ["Electronics", "Fashion", "Footwear"];
		document.write(categories.toString() +"<br>");
		document.write(categories.join("/"));
	</script>

- In modern programming we dont prefer loops bcz in js
- Generally loop required intializer(means where to start) ie. it requires a condition, you need a increment
- But iterator is software design pattern it read if there are value it stop if there are no value 
- Here we dont need to write intializer or condition and where to start where to stop 
- It will take care of that and iterator are more faster than loop.
- So, in js we mostly use iterator
- Iterator is read sequential elements.

Syntax: Map Iterator

	collection.map(function(value){
		// use value
	})

eg.
<script>
    var products = ["Mouse","keyboard","laptop","adaptor"];
    products.map(function(value){
        document.write(`<li>${value}</li>`);
    })
</script>


Syntax: forEach Iterator

	collection.forEach(function(value, index){
	// use index & value
	})

- By index we can delete item , if duplicate value are present we can manipulate according to index.

eg.

<script>
    var products = ["Mouse", "keyboard", "laptop", "adaptor"];
    products.forEach(function (value, index) {
        document.write(`<p>[${index}] ${value}</p>`);
    })
</script>

- You can use explicit iterators for reading array properties and elements.

	for .. in		It can read all properties of array.(properties means index number)
	for .. of		It can read all elements of array.
	
- in and of is operator

Syntax:

	for(var property in collection){

	}

	for(var value of collection){

	}


eg.

<script>
    var categories = ["Electronics", "Fashion", "Footwear"];

2. Filtering Elements

	find()		: It is an iterator that finds and returns only first occurrence value
				  that matches the given condition.

	filter()	: It is an iterator that returns all values that match given condition

	slice()		: It can returns the values between specified index.

Syntax:

	collection.find(function(value){		=> returns one value
		return value condition;
	});

	collection.filter(function(value){		=> returns an array [ ]
		return value condition;
	});

eg.

<script>
    var values = [34000, 44000, 65000, 57000, 85000];
    var result = values.find(function (val) {
        return val > 50000;
    });
    document.write(`find() -> ${result}<br>`);

    var resultArr = values.filter(function (val) {
        return val > 50000;
    })

    document.write(`filter() -> ${resultArr}<br>`);

    values.filter(function (val) {
        return val > 50000;
    }).map(function (val) {
        document.write(`filter() : map() -> ${val}<br>`);
    })
</script>
